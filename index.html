Personalized federated learning

Configure Personalized FL

Personalization Setup

Personalized FL is different from standard FL: clients share a common representation but keep local adaptation for their own distribution.

What Changes?

Global FL tries to build one model for everyone. Personalized FL accepts that hospitals, phones, sensors, or users may have different distributions.

The simulator shows a shared model core plus client-specific heads and reports both global and personalized quality.

Guide

How to Use FL Simulator

1. Select data and task

Choose a dataset. For tabular data, select target and independent variables. For image or sequence data, features are treated as tensors/windows.

2. Choose model and FL setup

Set clients, rounds, client fraction, local epochs, aggregation, dropout, and IID/non-IID level.

3. Run live simulation

The live page shows server-to-client communication, training phases, metrics, client distributions, and confusion matrix or residuals.

4. Interpret critically

More clients or deeper models do not always mean better FL. Non-IID data, dropout, and communication constraints can hurt convergence.
